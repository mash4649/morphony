## まだ不足がある場合

### 追加で詰めるべき点

1. **技術スタックの決定（必須・着手前に）**

   - パッケージマネージャ（pip/uv, npm/pnpm, go mod 等）
   - CLIフレームワーク（click, commander, cobra 等）
   - テストフレームワーク（pytest, vitest, go test 等）
   - この4点が決まれば Issue #1 の AC が一意に確定する

2. **永続化方法の方針（Issue #1 着手時に）**
   - 状態・チェックポイント・メモリの保存先（ファイルベースJSON / SQLite / 他）
   - Issue #1 の型定義は永続化方法に依存しないが、Issue #3〜#4 までに決める必要がある
   - 技術スタック決定後に自然に決まる範囲なので、Issue #1 の中で決定すれば十分

### ただし今は詰めなくてよい点

  Tool インターフェースで抽象化されているため、他の Issue に影響しない
- **Web検索APIの選定**: web_search ツール（Issue #12）着手時に決めれば十分。同上
- **メモリの類似検索アルゴリズム**: キーワードベースで MVP は十分と明記されており、
